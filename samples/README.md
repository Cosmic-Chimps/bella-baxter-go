# Bella Baxter — Go SDK Samples

Samples showing how to integrate Bella Baxter secrets into Go applications.

| Sample | Approach | Best for |
|--------|----------|---------|
| [01-dotenv-file](./01-dotenv-file/) | `bella secrets get -o .env` + godotenv | Simplest setup, scripts, CI/CD |
| [02-process-inject](./02-process-inject/) | `bella run -- go run .` | Any command, zero deps |
| [03-stdlib](./03-stdlib/) | SDK in `main()` → `Config` struct → handlers | stdlib `net/http`, chi, etc. |
| [04-gin](./04-gin/) | SDK in `main()` → `BellaMiddleware` → `gin.Context` | Gin (also adapts to Echo/Fiber) |
| [05-typed-secrets](./05-typed-secrets/) | Generated `AppSecrets` struct — typed, autocomplete | Any app that wants type-safe secrets |

---

## Which approach to choose?

```
Do you want zero Go SDK code?
├── YES → 01 (.env file) or 02 (bella run)
│
└── NO → Do you want typed, IDE-autocomplete secrets?
    ├── YES → 05-typed-secrets (generated AppSecrets struct)
    │
    └── NO → What framework?
        ├── stdlib / chi → 03-stdlib (Config struct passed to handlers)
        └── Gin / Echo / Fiber → 04-gin (middleware + context)
```

---

## Key difference from other languages

In Go, **all frameworks use the same pattern** — load secrets in `main()` before starting the server. There are no framework-specific startup hooks because Go's `main()` IS the startup hook. The only variation is how you pass secrets to handlers (struct receiver vs middleware vs context).

## Prerequisites

- Go 1.22+ (samples 03 and 04 use method-pattern routing introduced in Go 1.22)
- Bella Baxter account + API key

```bash
export BELLA_BAXTER_URL=https://baxter.yourcompany.com
export BELLA_API_KEY=bax-xxxxxxxxxxxxxxxxxxxx

# Self-hosted Bella Baxter? Also set the API URL:
# export BELLA_API_URL=https://your-bella-instance.com
export BELLA_ENV=production    # defaults to "production"
```

## Secret rotation support

| Sample | Automatic rotation | Notes |
|--------|-------------------|-------|
| `01-dotenv-file` | ❌ No | Re-run `bella secrets get -o .env` + restart |
| `02-process-inject` | ❌ No | Re-run `bella run --` to restart with fresh secrets |
| `03-stdlib` | ❌ No (manual possible) | Add `sync.RWMutex` + goroutine for live refresh |
| `04-gin` | ❌ No (manual possible) | Same goroutine pattern as `03-stdlib` |
| `05-typed-secrets` | ❌ No | Typed accessors call `os.Getenv` each time — restart picks up new values |

**Go does not have a built-in config reload hook** like .NET's `IChangeToken`. All patterns require either a process restart or an explicit polling goroutine with `sync.RWMutex`. See each sample's README for the goroutine pattern.

## Typed Secret Code Generation

`bella secrets generate go` fetches the secrets manifest (key names + type hints, no values) from the Bella API and generates a typed `AppSecrets` struct. Each method reads from `os.Getenv` at runtime — no secret values are ever embedded in the generated file.

```bash
bella secrets generate go \
  --project my-app \
  --environment production \
  --output app_secrets.go
```

**Generated `app_secrets.go`:**

```go
// Code generated by bella secrets generate go — do not edit manually.
package main

import (
"os"
"strconv"
)

type AppSecrets struct{}

func (s AppSecrets) DatabaseURL() string {
v := os.Getenv("DATABASE_URL")
if v == "" {
panic("secret 'DATABASE_URL' is not set")
}
return v
}

func (s AppSecrets) Port() int {
v := os.Getenv("PORT")
if v == "" {
panic("secret 'PORT' is not set")
}
n, err := strconv.Atoi(v)
if err != nil {
panic("secret 'PORT' is not a valid int: " + err.Error())
}
return n
}
```

### Usage alongside `bella run` or the SDK

```go
// Load secrets into the process first:
//   bella run -- go run .
//   or use the Go SDK in main()

secrets := AppSecrets{}
db  := secrets.DatabaseURL() // string, panics if missing
port := secrets.Port()       // int, parsed automatically
```

Because each method calls `os.Getenv` on every invocation, rotating a secret and restarting the process (via `bella run --watch`) picks up the new value automatically.

### Options

| Option | Default | Description |
|--------|---------|-------------|
| `-p, --project <slug>` | `.bella` context | Project slug |
| `-e, --environment <slug>` | `.bella` context | Environment slug |
| `--provider <slug>` | `default` | Provider slug |
| `-o, --output <path>` | `app_secrets.go` | Output file path |
| `--class-name <name>` | `AppSecrets` | Struct name |
| `--dry-run` | — | Print to stdout without writing |
